# AIGC 提示词设计师方法论

## 目的

这份文档定义“专业 AIGC 提示词设计师”的工作方法。

它承接 `概念设计师.md`：概念设计师负责确定画面要讲什么，AIGC 提示词设计师负责把结构化概念稳定转换成模型可执行的 prompt 和请求体，同时避免语义漂移、参数误用和批量风格失控。

## 核心定位

AIGC 提示词设计师不是把中文描述翻译成英文的人，也不是堆砌“高清、电影感、大师作品”的人。

专业提示词设计师要做的是：

- 保留概念的核心语义。
- 把抽象意图转成可视觉化的对象、空间、动作、材质和光线。
- 区分“提示词内容”和“模型参数”。
- 为不同生成模型选择合法配置。
- 控制批量生成的一致性和差异化。
- 给失败结果提供可诊断的修改方向。

## 提示词语言的特性

提示词是给 AI 看的语言。它表面上使用自然语言，但本质上更接近一种“视觉任务规格语言”。

专业提示词必须具备以下十个语言特性。

### 1. 指令性

提示词不是单纯描写，也不是文学散文。它必须告诉模型要生成什么类型的结果。

弱表达：

```text
一个少年看见仙门。
```

强表达：

```text
生成一张电影级写实东方玄幻概念设计图：背竹篓的凡人少年第一次看见云中仙门。
```

指令性要求 prompt 至少明确：

- 产出类型：概念设计图、角色设定图、环境图、战斗 keyframe、道具设定图。
- 核心主体：谁是画面第一主体。
- 任务目标：要表达什么叙事或审美目标。

### 2. 结构性

AI 更容易执行有层次的语义，而不是无序形容词堆叠。

推荐结构：

```text
产出类型，主体，叙事瞬间，场景，镜头，构图，暗示细节，光影色彩，材质画风，负向约束。
```

结构性要求每个信息块有清晰职责：

- 主体块解决“画谁”。
- 叙事块解决“发生什么”。
- 场景块解决“在哪里”。
- 镜头块解决“怎么看”。
- 构图块解决“怎么组织画面”。
- 风格块解决“画成什么质感”。
- 约束块解决“不要跑到哪里去”。

### 3. 可控性

提示词的专业价值不在于好听，而在于可控。

可控语言会把关键变量写清楚：

- 景别：远景、中景、近景、特写。
- 视角：平视、俯视、仰视、鸟瞰、低机位。
- 焦段：广角、标准镜头、长焦、macro。
- 景深：深景深、浅景深。
- 构图：中心构图、三分法、对角线、框景、留白、对称。
- 光线：逆光、侧光、顶光、轮廓光、冷暖对比。
- 色彩：低饱和、冷色调、暖色调、局部高亮。
- 材质：旧布、潮湿石壁、青铜锈、玉质、灰尘、泥土。

不可控语言通常是：

- 震撼
- 高级
- 好看
- 仙气
- 大师感
- 氛围拉满

这些词可以少量存在，但必须被具体视觉元素承载。

### 4. 语义密度

提示词应该用尽量少的词传递尽量多的可执行视觉信息。

高语义密度词会同时包含身份、时代、材质、场景和世界观。

例如：

- “破败的古修洞府”优于“一个旧地方”。
- “染血的储物袋和断裂飞剑”优于“战斗后的东西”。
- “云雾深处的宗门石阶”优于“一个漂亮的山”。

语义密度不是堆细节，而是选择能压缩世界观信息的词。

### 5. 视觉优先

图像模型只能生成可见内容。抽象概念必须翻译成视觉证据。

抽象表达：

```text
他很孤独。
```

视觉表达：

```text
人物独自站在巨大的空旷石阶中央，周围没有同伴，远处宗门灯火微弱，冷色月光拉长他的影子。
```

常见抽象词的视觉转译：

- 孤独：单人、小比例、大空间、长影、远处微弱灯火。
- 压迫：低机位、巨大遮挡物、高墙、阴影压顶。
- 危险：雾中轮廓、残骸、血迹、警觉姿态、未显露的威胁。
- 神秘：半遮挡、逆光、残碑、封印纹、局部光源。
- 凡人感：旧衣、泥路、粗糙器物、克制表情、身体疲惫。

### 6. 优先级

提示词里的信息不是平等的。越靠前、越明确、越与主体绑定的信息越容易被保留。

优先级顺序建议：

1. 产出类型和核心主体。
2. 叙事瞬间。
3. 场景和空间关系。
4. 镜头和构图。
5. 关键暗示细节。
6. 画风、光影、材质。
7. 负向约束。

如果模型总是丢失某个细节，不要只在尾部追加形容词，应把该细节提升到更靠前的位置，并绑定主体或动作。

### 7. 约束性

专业提示词同时包含正向目标和反向边界。

正向目标告诉模型要生成什么：

```text
电影级写实东方玄幻概念设计，古代中国器物和建筑基础，克制法术光效。
```

反向边界告诉模型不要生成什么：

```text
避免现代建筑、科幻机甲、西幻盔甲、二次元脸、页游特效、游戏 UI、文字、水印。
```

约束不是越多越好。负向约束应该排除最容易破坏项目统一性的错误方向。

### 8. 模型相关性

提示词不是跨模型完全一致的咒语。

不同模型可能在以下方面差异很大：

- 中文理解能力。
- 摄影语言理解能力。
- 对长 prompt 的保留能力。
- 对负向约束的响应能力。
- 对材质和真实感的偏好。
- 对尺寸、质量、数量、参考图等参数的支持方式。

因此提示词设计师必须结合 `lovart config`、`lovart plan`、`lovart quote` 的结果工作。

模型参数必须进入 request body，不能靠 prompt 猜：

- 尺寸
- 比例
- 质量档位
- 数量
- fast / relax / auto
- 输出格式

### 9. 可组合性

批量生成时，提示词应可以模块化组合。

推荐拆成：

- 项目级 prompt：统一世界观和画风。
- 分组级 prompt：统一图像类型。
- 单图级 prompt：表达当前概念。
- 负向 prompt：排除错误方向。

可组合性让 100 张图可以既统一又不重复。

错误做法是给 100 张图复制同一个“电影感、史诗、高清、国风”的通用尾巴，却没有为每张图建立清晰叙事差异。

### 10. 可验证性

专业提示词写完后必须能被检查。

验证问题：

- 模型是否知道第一主体是谁？
- 模型是否知道正在发生什么？
- 画面是否有可见的叙事证据？
- 镜头是否可执行？
- 构图是否不冲突？
- 风格是否和项目一致？
- 参数是否来自合法 `config.values`？
- 负向约束是否排除了主要跑偏方向？
- 生成失败后是否知道该改哪个字段？

如果一个 prompt 无法被诊断，它就不是专业 prompt。

## 提示词专家的十项能力

专业 AIGC 提示词设计师不是“会写漂亮句子”的人，而是能把概念、模型、参数、画面和批量策略连接起来的人。

### 1. 模型能力判断

提示词专家必须知道当前模型能做什么、不能做什么。

需要确认：

- 支持文生图、图生图、编辑、视频，还是多模态生成。
- 支持哪些尺寸、比例、质量、数量。
- 是否支持参考图、mask、音频、视频输入。
- 哪些字段是枚举值，哪些字段是自由输入。
- 哪些字段会影响价格、速度或权益。

在 Lovart 项目中，事实来源只能是：

```bash
lovart config <model>
lovart plan <model> --intent image-concept
lovart quote <model> --body-file request.json
```

### 2. 任务目标拆解

不同任务不能用同一种 prompt。

提示词专家要先判断任务类型：

- 角色关键帧：重身份、状态、动作、服饰道具、情绪。
- 场景环境图：重空间结构、尺度、历史痕迹、光线。
- 道具设定图：重材质、结构、功能线索、磨损。
- 战斗 keyframe：重动作瞬间、力量方向、动线、冲突。
- 生物概念：重体型比例、生态逻辑、行为和尺度关系。
- 主题海报：重象征关系、留白、强主题。

任务类型决定 prompt 的信息优先级。

### 3. 语义结构能力

提示词专家必须能把混乱想法拆成稳定结构。

最低结构：

- 核心主题
- 主体
- 叙事瞬间
- 场景
- 镜头
- 构图
- 暗示
- 画风
- 负向约束

这也是防止 prompt 漂移的基础。

### 4. 视觉转译能力

提示词专家必须把抽象目标翻译成可见画面。

转译方法：

- 情绪转物理证据。
- 世界观转器物和建筑。
- 身份转服饰、道具、姿态。
- 危险转环境痕迹和画外暗示。
- 成长转空间关系和尺度对比。

示例：

```text
“凡人面对仙道压迫”
```

不应直接写成：

```text
凡人很弱，仙道很强。
```

应转译为：

```text
衣着粗旧的少年站在巨大宗门石阶底部，石阶上方的修士只露出背影和衣摆，冷雾从阶缝流下，少年手中竹篓显得很小。
```

### 5. 镜头语言能力

提示词专家要知道镜头选择会改变叙事。

常用判断：

- 远景：强调人与世界的关系。
- 全景：展示动作和环境。
- 中景：平衡人物和场景。
- 近景：强调表情、手势、道具。
- 特写：强调细节、心理或线索。
- 广角：强调空间张力、压迫和尺度。
- 长焦：压缩空间，强调命运感、窥视感或人物隔阂。
- 俯视：弱化主体，表现渺小、命运、布局。
- 仰视：强化主体，表现压迫、神性或威胁。
- 低机位：适合表现巨物、山门、敌人、神像、异兽。
- 浅景深：突出主体，弱化环境。
- 深景深：适合世界观、环境叙事和大场景。

镜头不能和任务冲突。例如道具设定图通常不应使用极远景，世界观远景通常不应要求面部微表情。

### 6. 构图语言能力

提示词专家要知道构图如何控制阅读顺序。

常用构图母型：

- 中心构图：强调仪式感、主体权威、设定展示。
- 三分法：自然叙事，适合人物和环境平衡。
- 对称构图：宗门、祭坛、法阵、审判、秩序。
- 对角线构图：战斗、逃亡、坠落、追逐、冲突。
- 框景构图：洞府、门窗、树影、阵法边界，适合窥视和秘密。
- 引导线构图：石阶、河流、剑痕、光束，把视线引向主体。
- 留白构图：孤独、命运、未知、东方审美。
- 尺度对比：凡人与仙门、人物与巨兽、修士与天地。
- 前中后景构图：适合场景叙事和世界观深度。
- 剪影构图：适合神秘、英雄登场、身份未明。

构图必须服务叙事，不是为了显得专业而堆术语。

### 7. 场景叙事能力

提示词专家要让场景自己讲故事。

场景不只是背景，应该包含：

- 时间痕迹：破损、修补、风化、灰尘、苔藓。
- 事件痕迹：血迹、脚印、断剑、烧痕、残符。
- 文明痕迹：碑文、器物、建筑样式、阵法结构。
- 空间关系：入口、出口、高低、遮挡、路径。
- 画外暗示：未出现的敌人、刚离开的队伍、即将发生的危险。

好的场景 prompt 能让用户感觉这张图发生在一个真实世界里，而不是模型临时拼出的背景。

### 8. 风格控制能力

提示词专家必须能区分不同画风，而不是只写“高级感”。

常见概念设计风格：

- 电影级写实概念设计：适合关键帧、场景、角色气氛。
- 厚涂概念设计：适合早期探索，笔触感强。
- 写实国风玄幻：适合东方器物、宗门、修仙世界。
- 黑暗奇幻写实：适合魔道、禁地、邪修、异兽。
- 设定稿风格：适合道具、服饰、建筑、生物结构。
- 环境概念设计：适合空间、光线、尺度、世界观。
- 主题海报风格：适合强象征和主视觉。
- 电影 keyframe：适合叙事瞬间、动作、冲突。

风格控制要同时写：

- 主风格。
- 材质倾向。
- 光影倾向。
- 色彩倾向。
- 禁止风格。

### 9. 约束与反向约束能力

提示词专家要知道跑偏通常发生在哪里。

对“凡人修仙传”概念设计，常见跑偏方向包括：

- 二次元偶像脸。
- 页游仙侠特效。
- 西幻盔甲和城堡。
- 赛博、机甲、科幻武器。
- 过度发光、满屏粒子。
- 现代建筑、现代衣服、现代 UI。
- 没有叙事的站桩 pose。
- 过度华丽导致凡人感消失。

负向约束应该直接服务项目审美，不要无意义堆一长串通用禁词。

### 10. 批量一致性能力

生成 100 张图时，提示词专家要同时管理统一性和差异化。

统一性包括：

- 世界观统一。
- 画风统一。
- 材质真实度统一。
- 法术光效克制度统一。
- 时代器物统一。
- 审美边界统一。

差异化包括：

- 主体不同。
- 场景不同。
- 叙事瞬间不同。
- 镜头不同。
- 构图不同。
- 情绪不同。
- 暗示物不同。

批量提示词的目标不是让 100 张图“都很像”，而是让它们像同一个项目中的 100 张不同概念设计。

## 工作边界

提示词设计师负责：

- 从结构化概念中提取 prompt。
- 把核心、叙事瞬间、主体、场景、镜头、构图、暗示、画风合并为稳定描述。
- 根据模型能力选择合法参数。
- 为批量任务制定统一风格约束。
- 为每张图保留差异化重点。
- 生成可追踪、可复用、可迭代的 prompt 文档或 JSON 请求。

提示词设计师不应该：

- 擅自改写概念核心。
- 把模型不支持的尺寸、比例、质量值写进请求参数。
- 把 prompt 当成万能参数，把所有控制都塞进自然语言。
- 用没有依据的流行词堆叠代替画面设计。
- 在批量生成中让所有图共享同一站姿、同一镜头、同一构图。

## Prompt 与参数的分离

AIGC 生成请求通常包含两类信息：

### Prompt 内容

Prompt 负责表达画面语义，包括：

- 主体是谁。
- 正在发生什么。
- 场景在哪里。
- 镜头怎么拍。
- 构图如何组织。
- 暗示什么画外故事。
- 画风、材质、光线、气氛是什么。

### 模型参数

模型参数负责控制生成设置，包括：

- 模型名称。
- 尺寸或比例。
- 质量档位。
- 数量。
- 参考图。
- 输出格式。
- fast / relax / auto 模式。
- 是否允许付费。

这些参数必须来自工具返回的合法值。对于 Lovart 项目，agent 必须先使用：

```bash
lovart config <model>
lovart plan <model> --intent image-concept
lovart quote <model> --body-file request.json
```

提示词设计师不能把不存在于 `config.values` 的值写入请求参数。

## 结构化概念到 Prompt 的转换顺序

推荐固定顺序：

1. 主体与叙事瞬间。
2. 场景和世界观信息。
3. 镜头语言。
4. 构图语言。
5. 暗示细节。
6. 画风、材质、色彩和光线。
7. 禁止项或风格约束。

模板：

```text
主体与叙事瞬间，场景世界观细节，镜头语言，构图语言，暗示细节，画风与材质，风格约束，避免项。
```

示例：

```text
背竹篓的凡人少年刚离开村口，第一次看见云中若隐若现的仙门，泥路、炊烟、远山、脚边旧布包，24mm 广角远景，低机位轻微仰拍，深景深，尺度对比构图，少年很小而山门压在画面上方，电影写实东方玄幻，低饱和乡土色，克制法术光效，真实衣料和泥土材质，避免二次元、页游特效、过度发光。
```

## 结构化概念的无损转换原则

当输入已经是结构化概念时，提示词设计师的第一目标不是“写得更短”，而是“无损转换”。

尤其在批量概念设计中，`总画面描述` 只能作为沉浸式阅读入口，不能替代其他结构字段。每个字段都有独立职责：

- `核心` 决定这张图的主题，不可被总画面描述吞掉。
- `总画面描述` 提供完整画面感，是 prompt 的主体叙述基础。
- `叙事瞬间` 决定画面停在哪一秒，不可省略。
- `主体` 决定第一视觉优先级，不可被场景稀释。
- `场景` 提供世界观锚点和具体物件，不可只概括成“环境”。
- `镜头` 提供可拍摄性，不可只写“电影感”。
- `构图` 提供阅读顺序，不可只写“构图优秀”。
- `暗示` 提供画外故事和后续风险，不可被气氛词替代。
- `画风` 提供材质、色彩和渲染方向，不可只依赖项目级风格。

无损转换规则：

- 不得因为已有 `总画面描述` 就删除 `核心`、`叙事瞬间`、`主体`、`场景`、`镜头`、`构图`、`暗示`、`画风`。
- 不得把多个不同字段压缩成一句模糊总结，例如把 `暗示` 简化成“很危险”。
- 不得删除场景字段中的具体物件，例如“竹牌、照妖镜、阵旗、旧血痕、收费石碑”。
- 不得删除镜头参数中的焦段、景别、机位和景深，除非模型明确不理解该类摄影语言。
- 不得删除构图母型，例如“框中框、放射构图、环形构图、负空间构图”。
- 不得把 `画风` 中的材质、色彩、渲染方向完全交给项目级 prompt。
- 如果为了模型长度限制需要压缩，必须先生成无损版本，再生成压缩版本，并标记 `compressed=true`。

推荐无损单图 prompt 模板：

```text
生成一张电影级写实东方玄幻概念设计图。
核心叙事：{核心}
完整画面：{总画面描述}
叙事瞬间：{叙事瞬间}
第一主体：{主体}
场景世界观：{场景}
镜头语言：{镜头}
构图语言：{构图}
暗示叙事：{暗示}
画风材质：{画风}
```

这个模板比单段散文更适合 agent 批量调用，因为每个字段都能被检查、替换和追踪。

无损转换后的检查标准：

- 源概念中的 9 个字段是否都进入 prompt。
- `总画面描述` 是否保留沉浸感。
- `核心` 是否仍能单独读出来。
- `暗示` 是否仍是具体画外故事，而不是抽象情绪。
- `场景` 中的物件是否没有被大面积删除。
- `镜头` 和 `构图` 是否仍是可执行指令。
- `画风` 是否保留单图差异，而不是只使用项目级风格。

## Prompt 分层结构

复杂项目不应该只写一段 prompt。可以分成四层。

### 项目级提示词

所有图共享，用于统一世界观和画风。

示例：

```text
电影级写实东方玄幻概念设计，凡人流修仙气质，低饱和真实材质，克制法术光效，古代中国建筑和器物基础，厚涂概念笔触，强叙事性画面，避免二次元、廉价仙侠滤镜、过度发光、页游风。
```

### 分组级提示词

同一类图共享，用于区分角色、宗门、秘境、战斗、道具、生物、主题海报。

示例：

```text
环境气氛图，强调空间尺度、历史痕迹、雾、光线和未知危险，人物比例较小，场景本身讲述世界观。
```

### 单图级提示词

每张图独有，用于表达核心概念。

示例：

```text
上古洞府入口，藤蔓覆盖的石门缝隙透出微弱灵光，湿林、兽骨、残碑、冷雾，洞府像等待猎物的口。
```

### 负向约束

用于排除错误风格和错误画面。

示例：

```text
避免现代建筑、科幻机甲、二次元脸、过度发光、游戏 UI、廉价特效、卡通比例、塑料材质、无叙事站姿。
```

## 专业 Prompt 的质量标准

一个好 prompt 应该满足：

- 字段完整：结构化概念的核心、总画面、叙事瞬间、主体、场景、镜头、构图、暗示、画风都被保留。
- 可视化：每个抽象词都有画面承载物。
- 可拍摄：镜头位置、距离、景别明确。
- 可构图：画面阅读顺序明确。
- 可诊断：失败后知道改主体、场景、镜头、构图、风格还是参数。
- 可批量：同一项目内风格统一，但每张图有独立叙事。
- 不漂移：不改变原概念核心。

坏 prompt 常见问题：

- 只堆形容词：史诗、震撼、高清、大师、电影感。
- 没有叙事瞬间：只有设定，没有事件。
- 没有主体优先级：模型不知道先画谁。
- 没有场景逻辑：背景只是装饰。
- 镜头和构图冲突：例如极近特写同时要求完整城市全景。
- 总画面描述替代了全部字段，导致核心、暗示、画风、具体物件丢失。
- 参数混入 prompt：把应该由模型参数控制的尺寸、数量、输出格式写进自然语言。

## Prompt 设计流程

### 1. 读取结构化概念

先读取：

- 核心
- 叙事瞬间
- 主体
- 场景
- 镜头
- 构图
- 暗示
- 画风

不允许只读标题。

### 2. 判断图像类型

把概念归类：

- 角色关键帧
- 场景环境图
- 道具设定图
- 战斗 keyframe
- 生物概念
- 建筑设定
- 世界观远景
- 主题海报

不同类型使用不同 prompt 重点。

### 3. 生成单图 prompt

按固定顺序写成完整自然语言。

### 4. 生成参数请求体

调用模型配置：

```bash
lovart config <model>
```

只使用合法参数值。prompt 是自由输入字段，尺寸、质量、数量等是结构化参数。

### 5. 询价和安全检查

调用：

```bash
lovart quote <model> --body-file request.json
lovart generate <model> --body-file request.json --dry-run
```

真实生成前必须确认 quote 和 paid gate。

### 6. 结果诊断

如果结果不符合预期，按字段诊断：

- 主体错：强化主体位置、外观、动作。
- 场景错：增加环境锚点。
- 构图错：明确中心、三分、远景、近景、留白等。
- 风格错：调整项目级和分组级风格。
- 细节丢失：把暗示细节放到更靠前位置。
- 画面太乱：减少对象数量，提升主体优先级。

## 不同类型的 Prompt 模板

### 角色关键帧

```text
角色身份与状态，正在发生的叙事瞬间，服饰和道具，所在场景，镜头距离和角度，构图关系，情绪暗示，画风材质，避免项。
```

### 场景环境图

```text
地点名称，空间结构，时代和文明痕迹，前中后景层次，人物比例，光线和天气，暗示细节，环境概念设计风格，避免项。
```

### 道具设定图

```text
道具名称，材质结构，功能线索，使用痕迹，周围辅助物，近景或 macro 镜头，中心构图，真实材质渲染，避免项。
```

### 战斗 Keyframe

```text
冲突双方，动作发生的瞬间，力量方向，战场环境，镜头运动感，构图动线，战术暗示，电影 keyframe 风格，避免项。
```

### 生物概念

```text
生物类型，体型比例，生态环境，身体结构，行为瞬间，与人物的尺度关系，光线气氛，生物概念设计风格，避免项。
```

### 主题海报

```text
主题命题，象征主体，空间留白，明暗或尺度对比，暗示物，情绪色彩，象征主义写实海报风，避免项。
```

## 批量生成策略

批量生成不是把 100 个 prompt 机械丢进去。需要控制三件事：

### 统一性

统一：

- 世界观
- 时代质感
- 主画风
- 材质真实性
- 法术光效强度
- 避免项

### 差异化

每张至少变化一个主维度：

- 主体类型
- 场景类型
- 镜头焦段
- 构图母型
- 光线气氛
- 叙事瞬间
- 暗示物

### 可追踪性

每张图应保留编号和结构化来源：

- concept_id
- title
- prompt
- model
- config
- quote
- output_path
- revision_notes


## 中文与英文 Prompt

如果模型中文理解稳定，可以直接中文 prompt。

如果模型英文表现更稳定，推荐策略是：

- 保留中文结构化源文档。
- 自动生成英文 prompt。
- 不丢弃中文字段。
- 修改概念时只改中文源字段，再重新生成英文。

英文 prompt 不应自由发挥，而应忠实翻译结构字段。

## 与 Lovart 工具的协作规则

Agent 使用 Lovart 生成时，提示词设计师必须遵守：

- 先 `lovart setup`。
- 未指定模型时，先 `lovart plan --intent image-concept`。
- 固定模型时，先 `lovart config <model>`。
- 参数值只从 `config.values` 中选择。
- prompt 是自由输入，不能把 prompt 里的描述当成合法参数。
- 积分以 `lovart quote` 或 `plan.routes[].quote.exact=true` 为准。
- 真实生成默认只允许 0 积分；付费必须显式预算确认。

## 语义防漂移规则

- Prompt 只能表达结构化概念，不能替换结构化概念。
- Prompt 不能只保留 `总画面描述`，必须保留所有结构字段的独立语义。
- 如果结果需要更华丽，应改画风字段，不改核心字段。
- 如果结果需要更清楚，应改主体、镜头或构图字段。
- 如果结果需要更有故事，应改叙事瞬间或暗示字段。
- 如果结果需要更统一，应改项目级提示词。
- 如果结果需要更差异化，应改分组级或单图级字段。
- 如果 prompt 更好看但不再讲原来的故事，就是失败。

## Prompt 评审清单

生成前检查：

- 是否保留了源概念的全部字段？
- 是否没有把 `核心`、`暗示`、`画风` 压缩掉？
- 是否没有删除场景中的具体物件？
- 是否有明确主体？
- 是否有明确叙事瞬间？
- 是否有场景世界观信息？
- 是否有镜头语言？
- 是否有构图母型？
- 是否有暗示细节？
- 是否有画风和材质约束？
- 是否没有非法参数值？
- 是否没有把所有控制塞进 prompt？
- 是否能从结构化概念追溯回来？

生成后检查：

- 主体是否正确？
- 画面是否讲了原来的故事？
- 构图是否符合预期？
- 镜头距离是否正确？
- 暗示是否被保留？
- 风格是否统一？
- 是否有批量重复问题？
- 是否需要改 prompt 还是改模型参数？
