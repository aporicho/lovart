# AIGC 提示词设计师方法论

## 目的

这份文档定义“专业 AIGC 提示词设计师”的工作方法。

它承接 `概念设计师.md`：概念设计师负责确定画面要讲什么，AIGC 提示词设计师负责把结构化概念稳定转换成模型可执行的 prompt 和请求体，同时避免语义漂移、参数误用和批量风格失控。

## 核心定位

AIGC 提示词设计师不是把中文描述翻译成英文的人，也不是堆砌“高清、电影感、大师作品”的人。

专业提示词设计师要做的是：

- 保留概念的核心语义。
- 把抽象意图转成可视觉化的对象、空间、动作、材质和光线。
- 区分“提示词内容”和“模型参数”。
- 为不同生成模型选择合法配置。
- 控制批量生成的一致性和差异化。
- 给失败结果提供可诊断的修改方向。

## 工作边界

提示词设计师负责：

- 从结构化概念中提取 prompt。
- 把核心、叙事瞬间、主体、场景、镜头、构图、暗示、画风合并为稳定描述。
- 根据模型能力选择合法参数。
- 为批量任务制定统一风格约束。
- 为每张图保留差异化重点。
- 生成可追踪、可复用、可迭代的 prompt 文档或 JSON 请求。

提示词设计师不应该：

- 擅自改写概念核心。
- 把模型不支持的尺寸、比例、质量值写进请求参数。
- 把 prompt 当成万能参数，把所有控制都塞进自然语言。
- 用没有依据的流行词堆叠代替画面设计。
- 在批量生成中让所有图共享同一站姿、同一镜头、同一构图。

## Prompt 与参数的分离

AIGC 生成请求通常包含两类信息：

### Prompt 内容

Prompt 负责表达画面语义，包括：

- 主体是谁。
- 正在发生什么。
- 场景在哪里。
- 镜头怎么拍。
- 构图如何组织。
- 暗示什么画外故事。
- 画风、材质、光线、气氛是什么。

### 模型参数

模型参数负责控制生成设置，包括：

- 模型名称。
- 尺寸或比例。
- 质量档位。
- 数量。
- 参考图。
- 输出格式。
- fast / relax / auto 模式。
- 是否允许付费。

这些参数必须来自工具返回的合法值。对于 Lovart 项目，agent 必须先使用：

```bash
lovart config <model>
lovart plan <model> --intent image-concept
lovart quote <model> --body-file request.json
```

提示词设计师不能把不存在于 `config.values` 的值写入请求参数。

## 结构化概念到 Prompt 的转换顺序

推荐固定顺序：

1. 主体与叙事瞬间。
2. 场景和世界观信息。
3. 镜头语言。
4. 构图语言。
5. 暗示细节。
6. 画风、材质、色彩和光线。
7. 禁止项或风格约束。

模板：

```text
主体与叙事瞬间，场景世界观细节，镜头语言，构图语言，暗示细节，画风与材质，风格约束，避免项。
```

示例：

```text
背竹篓的凡人少年刚离开村口，第一次看见云中若隐若现的仙门，泥路、炊烟、远山、脚边旧布包，24mm 广角远景，低机位轻微仰拍，深景深，尺度对比构图，少年很小而山门压在画面上方，电影写实东方玄幻，低饱和乡土色，克制法术光效，真实衣料和泥土材质，避免二次元、页游特效、过度发光。
```

## Prompt 分层结构

复杂项目不应该只写一段 prompt。可以分成四层。

### 项目级提示词

所有图共享，用于统一世界观和画风。

示例：

```text
电影级写实东方玄幻概念设计，凡人流修仙气质，低饱和真实材质，克制法术光效，古代中国建筑和器物基础，厚涂概念笔触，强叙事性画面，避免二次元、廉价仙侠滤镜、过度发光、页游风。
```

### 分组级提示词

同一类图共享，用于区分角色、宗门、秘境、战斗、道具、生物、主题海报。

示例：

```text
环境气氛图，强调空间尺度、历史痕迹、雾、光线和未知危险，人物比例较小，场景本身讲述世界观。
```

### 单图级提示词

每张图独有，用于表达核心概念。

示例：

```text
上古洞府入口，藤蔓覆盖的石门缝隙透出微弱灵光，湿林、兽骨、残碑、冷雾，洞府像等待猎物的口。
```

### 负向约束

用于排除错误风格和错误画面。

示例：

```text
避免现代建筑、科幻机甲、二次元脸、过度发光、游戏 UI、廉价特效、卡通比例、塑料材质、无叙事站姿。
```

## 专业 Prompt 的质量标准

一个好 prompt 应该满足：

- 可视化：每个抽象词都有画面承载物。
- 可拍摄：镜头位置、距离、景别明确。
- 可构图：画面阅读顺序明确。
- 可诊断：失败后知道改主体、场景、镜头、构图、风格还是参数。
- 可批量：同一项目内风格统一，但每张图有独立叙事。
- 不漂移：不改变原概念核心。

坏 prompt 常见问题：

- 只堆形容词：史诗、震撼、高清、大师、电影感。
- 没有叙事瞬间：只有设定，没有事件。
- 没有主体优先级：模型不知道先画谁。
- 没有场景逻辑：背景只是装饰。
- 镜头和构图冲突：例如极近特写同时要求完整城市全景。
- 参数混入 prompt：把应该由模型参数控制的尺寸、数量、输出格式写进自然语言。

## Prompt 设计流程

### 1. 读取结构化概念

先读取：

- 核心
- 叙事瞬间
- 主体
- 场景
- 镜头
- 构图
- 暗示
- 画风

不允许只读标题。

### 2. 判断图像类型

把概念归类：

- 角色关键帧
- 场景环境图
- 道具设定图
- 战斗 keyframe
- 生物概念
- 建筑设定
- 世界观远景
- 主题海报

不同类型使用不同 prompt 重点。

### 3. 生成单图 prompt

按固定顺序写成完整自然语言。

### 4. 生成参数请求体

调用模型配置：

```bash
lovart config <model>
```

只使用合法参数值。prompt 是自由输入字段，尺寸、质量、数量等是结构化参数。

### 5. 询价和安全检查

调用：

```bash
lovart quote <model> --body-file request.json
lovart generate <model> --body-file request.json --dry-run
```

真实生成前必须确认 quote 和 paid gate。

### 6. 结果诊断

如果结果不符合预期，按字段诊断：

- 主体错：强化主体位置、外观、动作。
- 场景错：增加环境锚点。
- 构图错：明确中心、三分、远景、近景、留白等。
- 风格错：调整项目级和分组级风格。
- 细节丢失：把暗示细节放到更靠前位置。
- 画面太乱：减少对象数量，提升主体优先级。

## 不同类型的 Prompt 模板

### 角色关键帧

```text
角色身份与状态，正在发生的叙事瞬间，服饰和道具，所在场景，镜头距离和角度，构图关系，情绪暗示，画风材质，避免项。
```

### 场景环境图

```text
地点名称，空间结构，时代和文明痕迹，前中后景层次，人物比例，光线和天气，暗示细节，环境概念设计风格，避免项。
```

### 道具设定图

```text
道具名称，材质结构，功能线索，使用痕迹，周围辅助物，近景或 macro 镜头，中心构图，真实材质渲染，避免项。
```

### 战斗 Keyframe

```text
冲突双方，动作发生的瞬间，力量方向，战场环境，镜头运动感，构图动线，战术暗示，电影 keyframe 风格，避免项。
```

### 生物概念

```text
生物类型，体型比例，生态环境，身体结构，行为瞬间，与人物的尺度关系，光线气氛，生物概念设计风格，避免项。
```

### 主题海报

```text
主题命题，象征主体，空间留白，明暗或尺度对比，暗示物，情绪色彩，象征主义写实海报风，避免项。
```

## 批量生成策略

批量生成不是把 100 个 prompt 机械丢进去。需要控制三件事：

### 统一性

统一：

- 世界观
- 时代质感
- 主画风
- 材质真实性
- 法术光效强度
- 避免项

### 差异化

每张至少变化一个主维度：

- 主体类型
- 场景类型
- 镜头焦段
- 构图母型
- 光线气氛
- 叙事瞬间
- 暗示物

### 可追踪性

每张图应保留编号和结构化来源：

- concept_id
- title
- prompt
- model
- config
- quote
- output_path
- revision_notes

## 批量 Prompt JSON 建议结构

```json
{
  "concept_id": "001",
  "title": "凡人少年初入仙途",
  "prompt": "背竹篓的凡人少年刚离开村口...",
  "negative_prompt": "避免二次元、页游特效、过度发光...",
  "tags": ["character-keyframe", "xianxia", "cinematic-realism"],
  "source_fields": {
    "core": "凡人与仙门的第一次距离",
    "camera": "24mm 广角远景，低机位轻微仰拍，深景深",
    "composition": "尺度对比，少年很小，山门压在画面上方"
  }
}
```

## 中文与英文 Prompt

如果模型中文理解稳定，可以直接中文 prompt。

如果模型英文表现更稳定，推荐策略是：

- 保留中文结构化源文档。
- 自动生成英文 prompt。
- 不丢弃中文字段。
- 修改概念时只改中文源字段，再重新生成英文。

英文 prompt 不应自由发挥，而应忠实翻译结构字段。

## 与 Lovart 工具的协作规则

Agent 使用 Lovart 生成时，提示词设计师必须遵守：

- 先 `lovart setup`。
- 未指定模型时，先 `lovart plan --intent image-concept`。
- 固定模型时，先 `lovart config <model>`。
- 参数值只从 `config.values` 中选择。
- prompt 是自由输入，不能把 prompt 里的描述当成合法参数。
- 积分以 `lovart quote` 或 `plan.routes[].quote.exact=true` 为准。
- 真实生成默认只允许 0 积分；付费必须显式预算确认。

## 语义防漂移规则

- Prompt 只能表达结构化概念，不能替换结构化概念。
- 如果结果需要更华丽，应改画风字段，不改核心字段。
- 如果结果需要更清楚，应改主体、镜头或构图字段。
- 如果结果需要更有故事，应改叙事瞬间或暗示字段。
- 如果结果需要更统一，应改项目级提示词。
- 如果结果需要更差异化，应改分组级或单图级字段。
- 如果 prompt 更好看但不再讲原来的故事，就是失败。

## Prompt 评审清单

生成前检查：

- 是否有明确主体？
- 是否有明确叙事瞬间？
- 是否有场景世界观信息？
- 是否有镜头语言？
- 是否有构图母型？
- 是否有暗示细节？
- 是否有画风和材质约束？
- 是否没有非法参数值？
- 是否没有把所有控制塞进 prompt？
- 是否能从结构化概念追溯回来？

生成后检查：

- 主体是否正确？
- 画面是否讲了原来的故事？
- 构图是否符合预期？
- 镜头距离是否正确？
- 暗示是否被保留？
- 风格是否统一？
- 是否有批量重复问题？
- 是否需要改 prompt 还是改模型参数？
