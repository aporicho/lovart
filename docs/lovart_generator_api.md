## Authentication Headers Observed

The Lovart web client sends metadata requests and model requests to `https://lgw.lovart.ai` with these notable headers:

- `token`: browser login token
- `X-Send-Timestamp`: client timestamp after `/api/www/lovart/time/utc/timestamp` sync
- `X-Req-Uuid`: UUIDv4 without dashes
- `X-Client-Signature`: generated by the front-end WASM signer
- `Origin: https://www.lovart.ai` and `Referer: https://www.lovart.ai/`

The signer is reproduced by `lovart_reverse/signing/signature.js`; the package wraps it for signed LGW requests.

- `GET /v1/generator/schema?biz_type=16`: OpenAPI schema for model POST endpoints
- `POST /{provider}/{model}`: submit a model task, returns `TaskResponse`
